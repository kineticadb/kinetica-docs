<a id="inst-label"></a>


There are three methods for installing *Kinetica* on self-managed
infrastructure:

* *KAgent* -- The preferred method for installing *Kinetica*. *KAgent* automates
  the installation and configuration of *Kinetica*, rings (High Availability),
  and more.
* *Kubernetes* -- *Kinetica* can be installed to Kubernetes clusters using
  *Helm Charts* and the *Kinetica Operator for Kubernetes*
* *Manual* -- *Kinetica* can be installed manually via command line using common
  Linux-based package managers.


<Info>
Kinetica can be installed on pre-provisioned instances in AWS, Azure,
or GCP via KAgent.  For offerings provisioned within cloud environments
directly, see [Cloud-Ready](/content/cloud).
</Info>
