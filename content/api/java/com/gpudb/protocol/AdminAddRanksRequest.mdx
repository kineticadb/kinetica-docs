<div className="block">A set of parameters for <a href="/content/api/java/com/gpudb/GPUdb#adminAddRanks(com.gpudb.protocol.AdminAddRanksRequest)"><code>GPUdb.adminAddRanks</code></a>. <p> Add one or more ranks to an existing Kinetica cluster. The new ranks will not contain any data initially (other than replicated tables) and will not be assigned any shards. To rebalance data and shards across the cluster, use <a href="/content/api/java/com/gpudb/GPUdb#adminRebalance(com.gpudb.protocol.AdminRebalanceRequest)"><code>GPUdb.adminRebalance</code></a>. </p><p> The database must be offline for this operation, see <a href="/content/api/java/com/gpudb/GPUdb#adminOffline(com.gpudb.protocol.AdminOfflineRequest)"><code>GPUdb.adminOffline</code></a> </p><p> For example, if attempting to add three new ranks (two ranks on host 172.123.45.67 and one rank on host 172.123.45.68) to a Kinetica cluster with additional configuration parameters: </p><p> \* <a href="#getHosts()"><code>hosts</code></a> would be an array including 172.123.45.67 in the first two indices (signifying two ranks being added to host 172.123.45.67) and 172.123.45.68 in the last index (signifying one rank being added to host 172.123.45.67) </p><p> \* <a href="#getConfigParams()"><code>configParams</code></a> would be an array of maps, with each map corresponding to the ranks being added in <a href="#getHosts()"><code>hosts</code></a>. The key of each map would be the configuration parameter name and the value would be the parameter's value, e.g. '{"{"}"rank.gpu":"1"{"}"}' </p><p> This endpoint's processing includes copying all replicated table data to the new rank(s) and therefore could take a long time. The API call may time out if run directly. It is recommended to run this endpoint asynchronously via <a href="/content/api/java/com/gpudb/GPUdb#createJob(com.gpudb.protocol.CreateJobRequest)"><code>GPUdb.createJob</code></a>.</p></div>

<div className="dl notes">
<div className="dt">Parameters:</div>
<div className="dd"><code>hosts</code> - Array of host IP addresses (matching a hostN.address from the gpudb.conf file), or host identifiers (e.g. 'host0' from the gpudb.conf file), on which to add ranks to the cluster. The hosts must already be in the cluster. If needed beforehand, to add a new host to the cluster use <a href="/content/api/java/com/gpudb/GPUdb#adminAddHost(com.gpudb.protocol.AdminAddHostRequest)"><code>GPUdb.adminAddHost</code></a>. Include the same entry as many times as there are ranks to add to the cluster, e.g., if two ranks on host 172.123.45.67 should be added, <code>hosts</code> could look like '\["172.123.45.67", "172.123.45.67"\]'.
