<div className="block">
A set of parameters for <a href="/content/api/java/com/gpudb/GPUdb#adminRebalance(com.gpudb.protocol.AdminRebalanceRequest)"><code>GPUdb.adminRebalance</code></a>. 
<p> Rebalance the data in the cluster so that all nodes contain an equal number of records approximately and/or rebalance the shards to be equally distributed (as much as possible) across all the ranks. </p>
<p> The database must be offline for this operation, see <a href="/content/api/java/com/gpudb/GPUdb#adminOffline(com.gpudb.protocol.AdminOfflineRequest)"><code>GPUdb.adminOffline</code></a> </p>
<p> \* If <a href="/content/api/java/com/gpudb/GPUdb#adminRebalance(com.gpudb.protocol.AdminRebalanceRequest)"><code>GPUdb.adminRebalance</code></a> is invoked after a change is made to the cluster, e.g., a host was added or removed, <a href="/content/concepts/tables/#sharding" target="_top">sharded data</a> will be evenly redistributed across the cluster by number of shards per rank while unsharded data will be redistributed across the cluster by data size per rank </p>
<p> \* If <a href="/content/api/java/com/gpudb/GPUdb#adminRebalance(com.gpudb.protocol.AdminRebalanceRequest)"><code>GPUdb.adminRebalance</code></a> is invoked at some point when unsharded data (a.k.a. <a href="/content/concepts/tables/#random-sharding" target="_top">randomly-sharded</a>) in the cluster is unevenly distributed over time, sharded data will not move while unsharded data will be redistributed across the cluster by data size per rank </p>
<p> NOTE: Replicated data will not move as a result of this call </p>
<p> This endpoint's processing time depends on the amount of data in the system, thus the API call may time out if run directly. It is recommended to run this endpoint asynchronously via <a href="/content/api/java/com/gpudb/GPUdb#createJob(com.gpudb.protocol.CreateJobRequest)"><code>GPUdb.createJob</code></a>.</p>
