title: /admin/offline

{/* RST directive: role h2_style */}


```
URL: http://<db.host>:<db.port>/admin/offline
```
Take the system offline. When the system is offline, no user operations can be
performed with the exception of a system shutdown.

## Input Parameter Description

<ParamField body="offline" type="boolean">

Set to true if desired state is offline.

The supported values are:

- true
- false

</ParamField>

<ParamField body="options" type="map of string to strings">

Optional parameters.

The default value is an empty map ( \{\} ).

<Expandable title="options">

<ParamField body="flush_to_disk">

Flush to disk when going offline.

The supported values are:

- true
- false

</ParamField>

</Expandable>

</ParamField>


## Output Parameter Description

The Kinetica server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

<ResponseField name="status" type="String">

'OK' or 'ERROR'

</ResponseField>

<ResponseField name="message" type="String">

Empty if success or an error message

</ResponseField>

<ResponseField name="data_type" type="String">

'admin_offline_response' or 'none' in case of an error

</ResponseField>

<ResponseField name="data" type="String">

Empty string

</ResponseField>

<ResponseField name="data_str" type="JSON or String">

This embedded JSON represents the result of the /admin/offline endpoint:

<Expandable title="data_str">

<ResponseField name="is_offline" type="boolean">

Returns true if the system is offline, or false otherwise.

</ResponseField>

<ResponseField name="info" type="map of string to strings">

Additional information.

</ResponseField>

</Expandable>

Empty string in case of an error.

</ResponseField>
