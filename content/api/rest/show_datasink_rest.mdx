title: /show/datasink

{/* RST directive: role h2_style */}


```
URL: http://<db.host>:<db.port>/show/datasink
```
Shows information about a specified
[data sink](../../concepts/data_sinks/) or all data sinks.

## Input Parameter Description

<ParamField body="name" type="string">

Name of the data sink for which to retrieve information. The name must refer to a currently existing data sink. If '*' is specified, information about all data sinks will be returned.

</ParamField>

<ParamField body="options" type="map of string to strings">

Optional parameters.

The default value is an empty map ( \{\} ).

</ParamField>


## Output Parameter Description

The Kinetica server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

<ResponseField name="status" type="String">

'OK' or 'ERROR'

</ResponseField>

<ResponseField name="message" type="String">

Empty if success or an error message

</ResponseField>

<ResponseField name="data_type" type="String">

'show_datasink_response' or 'none' in case of an error

</ResponseField>

<ResponseField name="data" type="String">

Empty string

</ResponseField>

<ResponseField name="data_str" type="JSON or String">

This embedded JSON represents the result of the /show/datasink endpoint:

<Expandable title="data_str">

<ResponseField name="datasink_names" type="array of strings">

The data sink names.

</ResponseField>

<ResponseField name="destination_types" type="array of strings">

The destination type of the data sinks named in output parameter *datasink_names*.

</ResponseField>

<ResponseField name="additional_info" type="array of maps of string to strings">

Additional information about the respective data sinks in output parameter *datasink_names*.

<Expandable title="additional_info">

<ResponseField name="destination">

Destination for the output data in 'destination_type://path[:port]' format.

</ResponseField>

<ResponseField name="kafka_topic_name">

Kafka topic if the data sink type is a Kafka broker.

</ResponseField>

<ResponseField name="user_name">

Name of the remote system user.

</ResponseField>

</Expandable>

</ResponseField>

<ResponseField name="info" type="map of string to strings">

Additional information.

</ResponseField>

</Expandable>

Empty string in case of an error.

</ResponseField>
