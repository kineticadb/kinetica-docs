title: /show/datasource

{/* RST directive: role h2_style */}


```
URL: http://<db.host>:<db.port>/show/datasource
```
Shows information about a specified
[data source](../../concepts/data_sources/) or all data sources.

## Input Parameter Description

<ParamField body="name" type="string">

Name of the data source for which to retrieve information. The name must refer to a currently existing data source. If '*' is specified, information about all data sources will be returned.

</ParamField>

<ParamField body="options" type="map of string to strings">

Optional parameters.

The default value is an empty map ( \{\} ).

</ParamField>


## Output Parameter Description

The Kinetica server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

<ResponseField name="status" type="String">

'OK' or 'ERROR'

</ResponseField>

<ResponseField name="message" type="String">

Empty if success or an error message

</ResponseField>

<ResponseField name="data_type" type="String">

'show_datasource_response' or 'none' in case of an error

</ResponseField>

<ResponseField name="data" type="String">

Empty string

</ResponseField>

<ResponseField name="data_str" type="JSON or String">

This embedded JSON represents the result of the /show/datasource endpoint:

<Expandable title="data_str">

<ResponseField name="datasource_names" type="array of strings">

The data source names.

</ResponseField>

<ResponseField name="storage_provider_types" type="array of strings">

The storage provider type of the data sources named in output parameter *datasource_names*.

- **hdfs**: Apache Hadoop Distributed File System.
- **s3**: Amazon S3 bucket.

</ResponseField>

<ResponseField name="additional_info" type="array of maps of string to strings">

Additional information about the respective data sources in output parameter *datasource_names*.

<Expandable title="additional_info">

<ResponseField name="location">

Location of the remote storage in 'storage_provider_type://[storage_path[:storage_port]]' format.

</ResponseField>

<ResponseField name="s3_bucket_name">

Name of the Amazon S3 bucket used as the data source.

</ResponseField>

<ResponseField name="s3_region">

Name of the Amazon S3 region where the bucket is located.

</ResponseField>

<ResponseField name="hdfs_kerberos_keytab">

Kerberos key for the given HDFS user.

</ResponseField>

<ResponseField name="user_name">

Name of the remote system user.

</ResponseField>

</Expandable>

</ResponseField>

<ResponseField name="info" type="map of string to strings">

Additional information.

</ResponseField>

</Expandable>

Empty string in case of an error.

</ResponseField>
