title: /show/proc

{/* RST directive: role h2_style */}


```
URL: http://<db.host>:<db.port>/show/proc
```
Shows information about a proc.

## Input Parameter Description

<ParamField body="proc_name" type="string">

Name of the proc to show information about. If specified, must be the name of a currently existing proc. If not specified, information about all procs will be returned.

The default value is ''.

</ParamField>

<ParamField body="options" type="map of string to strings">

Optional parameters.

The default value is an empty map ( \{\} ).

<Expandable title="options">

<ParamField body="include_files">

If set to *true*, the files that make up the proc will be returned. If set to *false*, the files will not be returned.

The default value is `false`.

The supported values are:

- true
- false

</ParamField>

</Expandable>

</ParamField>


## Output Parameter Description

The Kinetica server embeds the endpoint response inside a standard response structure which contains status information and the actual response to the query.  Here is a description of the various fields of the wrapper:

<ResponseField name="status" type="String">

'OK' or 'ERROR'

</ResponseField>

<ResponseField name="message" type="String">

Empty if success or an error message

</ResponseField>

<ResponseField name="data_type" type="String">

'show_proc_response' or 'none' in case of an error

</ResponseField>

<ResponseField name="data" type="String">

Empty string

</ResponseField>

<ResponseField name="data_str" type="JSON or String">

This embedded JSON represents the result of the /show/proc endpoint:

<Expandable title="data_str">

<ResponseField name="proc_names" type="array of strings">

The proc names.

</ResponseField>

<ResponseField name="execution_modes" type="array of strings">

The execution modes of the procs named in output parameter *proc_names*.

- **distributed**: Distributed
- **nondistributed**: Nondistributed

</ResponseField>

<ResponseField name="files" type="array of maps of string to bytes">

Maps of the files that make up the procs named in output parameter *proc_names*.

</ResponseField>

<ResponseField name="commands" type="array of strings">

The commands (excluding arguments) that will be invoked when the procs named in output parameter *proc_names* are executed.

</ResponseField>

<ResponseField name="args" type="array of arrays of strings">

Arrays of command-line arguments that will be passed to the procs named in output parameter *proc_names* when executed.

</ResponseField>

<ResponseField name="options" type="array of maps of string to strings">

The optional parameters for the procs named in output parameter *proc_names*.

</ResponseField>

<ResponseField name="info" type="map of string to strings">

Additional information.

</ResponseField>

</Expandable>

Empty string in case of an error.

</ResponseField>
