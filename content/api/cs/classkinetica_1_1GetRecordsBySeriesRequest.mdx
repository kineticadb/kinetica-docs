<h2 className="groupheader">Detailed Description</h2>
<div className="textblock">
<p>A set of parameters for <a className="el" href="">Kinetica.getRecordsBySeries</a>. </p>
<p>Retrieves the complete series/track records from the given <a className="el" href="#a63f0820942422b54bc11b786997324dd" title="Name of the table containing the complete series/track information to be returned for the tracks pres...">GetRecordsBySeriesRequest.world\_table\_name</a> based on the partial track information contained in the <a className="el" href="#a15eb465dc6f5f2a46a4525f9ea81efc9" title="Name of the table or view for which series/tracks will be fetched, in [schema_name.">GetRecordsBySeriesRequest.table\_name</a>.</p>
<p>This operation supports paging through the data via the <a className="el" href="#ad605d55c5e86cc87255fc5d48bd86d26" title="A positive integer indicating the number of initial series/tracks to skip (useful for paging through ...">GetRecordsBySeriesRequest.offset</a> and <a className="el" href="#a8cf4ebd8bf94026f763ba6e3868eb903" title="A positive integer indicating the maximum number of series/tracks to be returned.">GetRecordsBySeriesRequest.limit</a> parameters.</p>
<p>In contrast to <a className="el" href="">Kinetica.getRecords</a> this returns records grouped by series/track. So if <a className="el" href="#ad605d55c5e86cc87255fc5d48bd86d26" title="A positive integer indicating the number of initial series/tracks to skip (useful for paging through ...">GetRecordsBySeriesRequest.offset</a> is 0 and <a className="el" href="#a8cf4ebd8bf94026f763ba6e3868eb903" title="A positive integer indicating the maximum number of series/tracks to be returned.">GetRecordsBySeriesRequest.limit</a> is 5 this operation would return the first 5 series/tracks in <a className="el" href="#a15eb465dc6f5f2a46a4525f9ea81efc9" title="Name of the table or view for which series/tracks will be fetched, in [schema_name.">GetRecordsBySeriesRequest.table\_name</a>. Each series/track will be returned sorted by their TIMESTAMP column.</p>
