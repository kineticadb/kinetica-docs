<h2 className="groupheader">Detailed Description</h2>
<div className="textblock">
<p>A set of parameters for <a className="el" href="/content/api/cs/classkinetica_1_1Kinetica#a92656d0210734075bf9ea63867324123">Kinetica.filterByRange</a>. </p>
<p>Calculates which objects from a table have a column that is within the given bounds. An object from the table identified by <a className="el" href="#a602cc79c2269c107e85df95fca63dbdb" title="Name of the table on which the filter by range operation will be performed, in [schema_name.">FilterByRangeRequest.table\_name</a> is added to the view <a className="el" href="#a81921111393dd5570872eeba7894a608" title="If provided, then this will be the name of the view containing the results, in [schema_name.">FilterByRangeRequest.view\_name</a> if its column is within \[<a className="el" href="#ae6ecae3fa43f7de4d5da28c5e5a7df04" title="Value of the lower bound (inclusive).">FilterByRangeRequest.lower\_bound</a>, <a className="el" href="#a68dd4f9a3131ad6124d3d22ad09582fc" title="Value of the upper bound (inclusive).">FilterByRangeRequest.upper\_bound</a>\] (inclusive). The operation is synchronous. The response provides a count of the number of objects which passed the bound filter. Although this functionality can also be accomplished with the standard filter function, it is more efficient.</p>
<p>For track objects, the count reflects how many points fall within the given bounds (which may not include all the track points of any given track).</p>
