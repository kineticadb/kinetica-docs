<h2 className="groupheader">Detailed Description</h2>
<div className="textblock">
<p>A set of parameters for <a className="el" href="/content/api/cpp/classgpudb_1_1GPUdb#abaf7eca0d3c131b8fede4850d51f1a1a">GPUdb::createType</a>. </p>
<p>Creates a new type describing the columns of a table. The type definition is specified as a list of columns, each specified as a list of the column name, data type, and any column attributes.</p>
<p>Example of a type definition with some parameters: </p>
<div className="fragment">
<div className="line">\[</div>
<div className="line"> \[<span className="stringliteral">&quot;id&quot;</span>, <span className="stringliteral">&quot;int8&quot;</span>, <span className="stringliteral">&quot;primary\_key&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;dept\_id&quot;</span>, <span className="stringliteral">&quot;int8&quot;</span>, <span className="stringliteral">&quot;primary\_key&quot;</span>, <span className="stringliteral">&quot;shard\_key&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;manager\_id&quot;</span>, <span className="stringliteral">&quot;int8&quot;</span>, <span className="stringliteral">&quot;nullable&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;first\_name&quot;</span>, <span className="stringliteral">&quot;char32&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;last\_name&quot;</span>, <span className="stringliteral">&quot;char64&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;salary&quot;</span>, <span className="stringliteral">&quot;decimal&quot;</span>\],</div>
<div className="line"> \[<span className="stringliteral">&quot;hire\_date&quot;</span>, <span className="stringliteral">&quot;date&quot;</span>\]</div>
<div className="line">\]</div>
</div>
<p> Each column definition consists of the column name (which should meet the standard <a href="/content/concepts/tables/#table-naming-criteria" target="_top">column naming criteria</a>), the column's <a href="/content/concepts/types/#types-chart" target="_top">specific type</a> (int, long, float, double, string, bytes, or any of the possible values for <a className="el" href="#a51d65ae072fd0bbb9991dc83d7e0a7c6">properties</a>), and any <a href="/content/concepts/types/#types-data-handling" target="_top">data handling</a>, <a href="/content/concepts/types/#types-data-keys" target="_top">data key</a>, or <a href="/content/concepts/types/#types-data-replace" target="_top">data replacement</a> properties.</p>
<p>Note that some properties are mutually exclusive&ndash;i.e. they cannot be specified for any given column simultaneously. One example of mutually exclusive properties are <a className="el" href="/content/api/cpp/namespacegpudb#a8bf3bb8e3db95397e8b3992f78ebf740">primary\_key</a> and <a className="el" href="/content/api/cpp/namespacegpudb#a89eab971915fdca45d8d31771b2c291a">nullable</a>.</p>
<p>A single <a href="/content/concepts/tables/#primary-keys" target="_top">primary key</a> and/or single <a href="/content/concepts/tables/#shard-keys" target="_top">shard key</a> can be set across one or more columns. If a primary key is specified, then a uniqueness constraint is enforced, in that only a single object can exist with a given primary key column value (or set of values for the key columns, if using a composite primary key). When <a className="el" href="/content/api/cpp/classgpudb_1_1GPUdb#ab6ab327611452cc777c159d202146997">inserting</a> data into a table with a primary key, depending on the parameters in the request, incoming objects with primary key values that match existing objects will either overwrite (i.e. update) the existing object or will be skipped and not added into the set. </p>
<p className="definition">Definition at line <a className="el" href="/content/api/cpp/create__type_8h_source#l00063">63</a> of file <a className="el" href="/content/api/cpp/create__type_8h_source">create\_type.h</a>.</p>
</div>
