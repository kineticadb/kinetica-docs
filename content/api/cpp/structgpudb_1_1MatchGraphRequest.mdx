<h2 className="groupheader">Detailed Description</h2>
<div className="textblock">
<p>A set of parameters for <a className="el" href="/content/api/cpp/classgpudb_1_1GPUdb#aa906b4367c2f812aabd089ad7d0d8f3b">GPUdb::matchGraph</a>. </p>
<p>Matches a directed route implied by a given set of latitude/longitude points to an existing underlying road network graph using a given solution type.</p>
<p>IMPORTANT: It's highly recommended that you review the <a href="/content/graph_solver/network_graph_solver/" target="_top">Graphs and Solvers</a> concepts documentation, the <a href="/content/guides/graph_rest_guide/" target="_top">Graph REST Tutorial</a>, and/or some <a href="/content/guide-tags/graph---match/" target="_top">/match/graph examples</a> before using this endpoint. </p>

<td className="paramdir">\[in\]</td>
<td className="paramname">solveMethod\_</td>
<td>
The type of solver to use for graph matching. Supported values: 
<ul>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a7ad0718b76016b8c9b0a0de88ac84c7d">match\_graph\_markov\_chain</a>: Matches <span className="em arg">samplePoints\_</span> to the graph using the Hidden Markov Model (HMM)-based method, which conducts a range-tree closest-edge search to find the best combinations of possible road segments (<a className="el" href="/content/api/cpp/namespacegpudb#a51c5de5ff0dfc27dbf854ef34c2b0c22">num\_segments</a>) for each sample point to create the best route. The route is secured one point at a time while looking ahead <a className="el" href="/content/api/cpp/namespacegpudb#a08b6ea9775a4bb3dc42b9eb705d02426">chain\_width</a> number of points, so the prediction is corrected after each point. This solution type is the most accurate but also the most computationally intensive. Related options: <a className="el" href="/content/api/cpp/namespacegpudb#a51c5de5ff0dfc27dbf854ef34c2b0c22">num\_segments</a> and <a className="el" href="/content/api/cpp/namespacegpudb#a08b6ea9775a4bb3dc42b9eb705d02426">chain\_width</a>. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a7e679642af2dff0fbc9ca0a45e2f1952">match\_graph\_match\_od\_pairs</a>: Matches <span className="em arg">samplePoints\_</span> to find the most probable path between origin and destination pairs with cost constraints. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a8f80b8e394deafc5d34236b69d6f9b73">match\_graph\_match\_supply\_demand</a>: Matches <span className="em arg">samplePoints\_</span> to optimize scheduling multiple supplies (trucks) with varying sizes to varying demand sites with varying capacities per depot. Related options: <a className="el" href="/content/api/cpp/namespacegpudb#a5ca9626de28542759a1b6293be8189f7">partial\_loading</a> and <a className="el" href="/content/api/cpp/namespacegpudb#a6bf775bf9a24a15eac91bccc03962882">max\_combinations</a>. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a0f0ad2148c66745ae0326d8ffa1feeda">match\_graph\_match\_batch\_solves</a>: Matches <span className="em arg">samplePoints\_</span> source and destination pairs for the shortest path solves in batch mode. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a870028a40aeeb32489b1ed2ac124ea51">match\_graph\_match\_loops</a>: Matches closed loops (Eulerian paths) originating and ending at each graph node within min and max hops (levels). </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#ace6c6283dc6cd2d9e29afea68c79b942">match\_graph\_match\_charging\_stations</a>: Matches an optimal path across a number of ev-charging stations between source and target locations. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a6f36f390b7dad827b2adbc627ee8a057">match\_graph\_match\_similarity</a>: Matches the intersection set(s) by computing the Jaccard similarity score between node pairs. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a6ed1924b6d396735845214ea5312acad">match\_graph\_match\_pickup\_dropoff</a>: Matches the pickups and dropoffs by optimizing the total trip costs. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a93ed321a55a284d86e5f0a755dea35d8">match\_graph\_match\_clusters</a>: Matches the graph nodes with a cluster index using Louvain clustering algorithm. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#ad5033e349cb6279334df27d2217259a1">match\_graph\_match\_pattern</a>: Matches a pattern in the graph. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#aa7d68e4514106fd078269ca284fcaeb3">match\_graph\_match\_embedding</a>: Creates vector node embeddings. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#ace7e2fec0b9a36ad6466bd61daabd542">match\_graph\_match\_isochrone</a>: Solves for isochrones for a set of input sources. </li>
<li> <a className="el" href="/content/api/cpp/namespacegpudb#a99341caade4262a70954f23065993c3c">match\_graph\_match\_route\_detour</a>: Computes detour costs for nearby stations at a mark point along each source-target route. </li>
</ul>
 The default value is <a className="el" href="/content/api/cpp/namespacegpudb#a7ad0718b76016b8c9b0a0de88ac84c7d">match\_graph\_markov\_chain</a>. 
</td>
</tr>

<td className="paramdir">\[in\]</td>

<td>The name of the table used to store the results, in \[schema\_name.\]table\_name format, using standard <a href="/content/concepts/tables/#table-name-resolution" target="_top">name resolution rules</a> and meeting <a href="/content/concepts/tables/#table-naming-criteria" target="_top">table naming criteria</a>.
